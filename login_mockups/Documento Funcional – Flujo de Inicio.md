**Documento Funcional – Flujo de Inicio y Escritorio Simulado**



**1. Introducción**



Este documento describe el comportamiento y la experiencia esperada del flujo de inicio de un sistema simulado, inspirado en la experiencia de un sistema operativo de escritorio.

El objetivo es guiar al equipo de implementación en la construcción de los mockups, asegurando consistencia visual, claridad en la navegación y una experiencia intuitiva para el usuario final.



El flujo no contempla autenticación real ni validación de credenciales. Todas las interacciones corresponden a una simulación visual y funcional.



**2. Alcance del Proyecto**



* Incluye



Pantalla de inicio (tipo pantalla bloqueada).



Pantalla de selección de usuario.



Escritorios personalizados por usuario.



Navegación completa entre pantallas.



Comportamientos básicos de interacción y respuesta visual.



* No incluye



Lógica de autenticación real.



Gestión de usuarios o seguridad.



Persistencia de sesión.



Funcionalidades reales de aplicaciones del escritorio.





**3. Flujo General de Usuario**



El usuario accede a la pantalla de inicio.



Desde la pantalla de inicio, avanza a la selección de usuario.



Selecciona uno de los perfiles disponibles.



Ingresa al escritorio correspondiente a ese perfil.



Desde el escritorio puede cerrar sesión y volver a la pantalla de inicio.







**4. Pantalla de Inicio**

**4.1. Propósito**



Presentar el punto de entrada al sistema simulado y comunicar visualmente la idea de un sistema operativo antes del acceso del usuario.



**4.2. Descripción Visual**



Pantalla a tamaño completo.



Imagen de fondo principal.



Elemento central tipo tarjeta semitransparente.



Dentro de la tarjeta:



Identidad visual del sistema (logo).



Imagen de perfil.



Input box con el texto “Contraseña” o “Iniciar sesión”, acompañado de un botón con un icono de flecha.



**4.3. Comportamiento**



El botón principal es el único punto de interacción.



Al interactuar con el botón, el sistema navega a la pantalla de selección de usuario.



No se solicita información al usuario.



**4.4. Consideraciones de Experiencia**



El botón debe ser claramente reconocible como interactivo.



Debe existir retroalimentación visual al pasar el cursor y al hacer clic.



La pantalla debe transmitir simplicidad y enfoque en una sola acción.



**5. Pantalla de Selección de Usuario**

**5.1. Propósito**



Permitir al usuario elegir el perfil con el cual accederá al sistema simulado.



**5.2. Descripción Visual**



Mantiene el fondo de pantalla para continuidad visual.



Bloque informativo central que muestra:



Nombre del sistema.



Hora actual destacada.



Fecha en formato legible.



Listado de perfiles de usuario:



Dos perfiles visibles.



Cada perfil incluye imagen y nombre.



Diseño tipo tarjeta o botón.



**5.3. Perfiles Disponibles**



Johnny



Shakira



**5.4. Comportamiento**



Cada perfil es interactivo.



Al seleccionar un perfil, el sistema carga directamente el escritorio correspondiente.



No existen pasos intermedios ni confirmaciones adicionales.



5.5. Consideraciones de Experiencia



Debe ser evidente que los perfiles son seleccionables.



Los perfiles deben reaccionar visualmente al pasar el cursor o recibir foco.



El diseño debe mantener jerarquía clara entre información y acciones.



**6. Escritorios por Usuario**

**6.1. Propósito**



Simular el entorno principal del sistema operativo para cada usuario seleccionado.



**6.2. Características Generales**



Pantalla completa.



Fondo visual tipo escritorio.



Íconos o accesos directos simulados.



Elementos visuales que refuercen la idea de sistema operativo.



**6.3. Diferenciación por Usuario**



Cada usuario debe contar con un escritorio claramente distinto.

Las diferencias pueden reflejarse en:



Imagen de fondo.



Paleta de colores.



Distribución de los elementos.



Tipo de íconos o contenido simulado.



6.4. Salida del Escritorio



Debe existir una acción visible para:



“Cerrar sesión” o



“Volver al inicio”.



Al usar esta acción, el sistema regresa a la pantalla de inicio.



**7. Navegación y Comportamiento General**

**7.1. Navegación**



La navegación entre pantallas debe ser directa y predecible.



No deben existir rutas confusas ni pantallas sin salida.



El recorrido debe seguir siempre el flujo definido.



**7.2. Transiciones**



Las transiciones deben sentirse suaves y naturales.



No son obligatorias animaciones complejas.



El foco principal es la claridad del flujo.



**8. Interacción y Usabilidad**

**8.1. Retroalimentación Visual**



Todos los elementos interactivos deben responder visualmente.



El usuario debe identificar fácilmente:



Qué puede seleccionar.



Qué acción está realizando.



**8.2. Accesibilidad Básica**



Los textos deben ser legibles sobre los fondos.



Los botones deben tener tamaño adecuado para interacción.



La experiencia debe ser usable con mouse y teclado.



**8.3. Adaptabilidad**



El diseño debe adaptarse a diferentes resoluciones de pantalla.



No debe romperse en pantallas más pequeñas.



Los elementos deben reacomodarse sin perder jerarquía visual.



**9. Criterios Generales** 

El flujo completo puede recorrerse sin bloqueos.



Cada pantalla cumple su propósito claramente.



Los usuarios Johnny y Shakira presentan experiencias diferenciadas.



Siempre existe una forma clara de regresar al inicio.



La experiencia se percibe coherente, fluida y consistente con los mockups.
